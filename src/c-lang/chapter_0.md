# chapter 0 ようこそC言語の世界へ

## 0.1 ようこそC言語の世界へ

C 言語は、見た目が少し無骨です。最初は `;` や `{}` や `*` が多く、きれいな魔法のようには見えないかもしれません。けれども C のよさは、その素朴さにあります。  
C は、コンピュータの動きをかなり近い位置から扱える一方で、必要な道具立ては比較的少なく、言語の中心は驚くほど小さいのです。

C を学ぶと、次のような力が身につきます。

- プログラムが「命令の列」であることを理解できる
- コンパイルという仕組みを知ることができる
- 変数、型、関数、配列、文字列、ポインタの基本を学べる
- ほかの多くの言語を学ぶときにも役立つ考え方が身につく

C は、どんな機能も自動で用意してくれる「便利な全部入り」ではありません。その代わり、何が起きているかを理解しやすい言語でもあります。だからこそ、最初は少し手間に感じても、基礎を越えた先で大きな力になります。

## 0.2 はじめてのプログラミング

まずは、いちばん有名な最初のプログラムを書いてみましょう。  
次のコードは、画面に文字を表示するだけの、最小限の C プログラムです。

```c
#include <stdio.h>

int main(void) {
    printf("Hello, world!\n");
    return 0;
}
```

この時点で細部を全部理解できなくても問題ありません。ここで見たいのは、**プログラムは文字の集まりとして書き、コンピュータはそれを読み、実行する**という基本の流れです。

- `#include <stdio.h>` は、`printf` を使うための準備です
- `int main(void)` は、プログラムの入口です
- `printf(...)` は、文字を表示します
- `return 0;` は、プログラムが正常に終わったことを示します

はじめは「書いたら動いた」という感覚が大切です。理解はあとから追いつきます。  
最初の目標は、**自分で少し書き換えて実行できること**です。例えば `Hello, world!` を自分の名前に変えてみてください。表示が変われば、もう立派にプログラムを動かしています。

## 0.3 まずは失敗してみる

C の学習では、エラーを恐れないことが重要です。  
たとえば次のように、`";"` を忘れるとコンパイラは文句を言います。

```c
#include <stdio.h>

int main(void) {
    printf("Hello\n")
    return 0;
}
```

このとき大切なのは、エラーを「失敗」として終わらせないことです。  
エラーメッセージは、コンピュータが出してくれるヒントです。最初は読みにくく感じても、少しずつ次のように考えましょう。

- どの行を指しているのか
- その少し前に書き忘れがないか
- 括弧 `()` や波かっこ `{}` が対応しているか

C の学習は、きれいに一発で成功することよりも、**失敗から原因を見つける練習**でもあります。

## 0.4 第0章のまとめ

- C は、コンピュータに近いところを扱える言語です
- 最初の目標は、理解しきれなくても「書いて動かす」ことです
- エラーは失敗ではなく、直すための手がかりです
- 本書では、古い習慣よりも現在の標準に沿った書き方を優先します
