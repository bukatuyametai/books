# chapter 1 プログラムの書き方

## 1.1 C言語プログラムの基礎知識

C のプログラムは、**コンピュータに順番に読ませる命令書**です。  
人間が読むときは見出しや余白が大切ですが、コンピュータにとっては、決められた文法どおりに書かれているかが何より重要です。

C の基本で覚えたいのは、次の 3 つです。

- **文**: `;` で終わるひとまとまりの命令
- **ブロック**: `{}` で囲まれた複数の文の集まり
- **関数**: ひとまとまりの処理に名前をつけたもの

たとえば次のコードでは、`printf` という関数を使って表示しています。

```c
#include <stdio.h>

int main(void) {
    printf("C言語へようこそ\n");
    return 0;
}
```

C は大文字と小文字を区別します。`printf` と `Printf` は別物です。  
また、スペースや改行は見た目を整えるためのもので、ほとんどの場合は文法上の意味を持ちません。ただし、読みやすく書くことはとても大切です。**人間に読みやすいコードは、あとで直しやすいコード**でもあります。

## 1.2 C言語プログラムの基本構造

C プログラムには、最初に知っておくと安心な決まりがあります。

```c
#include <stdio.h>

int main(void) {
    /* ここに処理を書く */
    return 0;
}
```

- `#include <stdio.h>` は、標準入出力の道具を使うための宣言です
- `main` はプログラムの入口で、ここから実行が始まります
- `return 0;` は「正常終了」を表します

`main` の `void` は「引数を受け取らない」ことを示します。初心者のうちは、まず `int main(void)` を基本形として覚えておけば十分です。

コメントも大切です。`/* ... */` で囲まれた部分は、コンパイルの際に処理されません。

```c
#include <stdio.h>

int main(void) {
    /* これは説明用のコメント */
    printf("コメントは実行されません\n");
    return 0;
}
```

コメントは「何をしているか」よりも、「なぜそうしているか」を書くと役に立ちます。コードを読めばわかることを、コメントで言い直しすぎないのがコツです。

## 1.3 第1章のまとめ

- C の基本単位は文とブロックです
- `main` はプログラムの入口です
- `#include` は外部の道具を使う準備です
- コメントは人間のために書きます
