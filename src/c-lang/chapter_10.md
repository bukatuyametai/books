# chapter 10 メモリアクセスのからくり

## 10.1 逃げられない理由

C では、メモリを自分で意識しなければならない場面が多くあります。  
それは面倒でもありますが、同時に大きな自由でもあります。自由には責任がつきものです。C が「強い」と言われるのは、メモリを細かく扱えるからです。

## 10.2 3つのからくり構造

メモリの扱いを理解するときは、まず大きく 3 つを考えるとわかりやすいです。

- **自動変数**: 関数の中で作られ、関数を抜けると役目を終える
- **静的領域**: プログラム全体で長く生きる
- **動的領域**: `malloc` などで必要な分だけ確保する

```c
int global_value = 1;   /* 静的領域 */
void f(void) {
    int local = 2;      /* 自動変数 */
}
```

## 10.3 2つのメモリアクセス手段

C のメモリアクセスには、主に「直接アクセス」と「間接アクセス」があります。  
変数名で直接読むのが直接アクセス、ポインタを通して読むのが間接アクセスです。

```c
int x = 10;
int *p = &x;

printf("%d\n", x);   /* 直接 */
printf("%d\n", *p);  /* 間接 */
```

どちらが良い悪いではなく、状況によって使い分けます。

## 10.4 自由なメモリアクセスの代償

C では、配列の端を超えて読んだり、すでに解放した領域を触ったりすると、深刻な不具合になります。  
これはコンパイラがすべて止めてくれるわけではありません。だからこそ、境界の確認が大切です。

- 範囲を超えない
- `malloc` したら `free` する
- 使い終わったポインタをそのままにしない

この 3 つを習慣にするだけでも、安全性はかなり上がります。

## 10.5 メモリを扱う標準関数

C の標準ライブラリには、メモリ操作の関数があります。

- `malloc`: メモリを確保する
- `calloc`: ゼロ初期化して確保する
- `realloc`: サイズを変える
- `free`: 解放する
- `memcpy`: メモリをコピーする
- `memset`: メモリを埋める

```c
int *p = malloc(3 * sizeof(int));
if (p != NULL) {
    p[0] = 1;
    p[1] = 2;
    p[2] = 3;
    free(p);
}
```

`malloc` の戻り値は `NULL` のこともあるので、確保に成功したか確認する習慣をつけましょう。

## 10.6 ヒープの利用

動的に確保したメモリは、必要な大きさを実行時に決められるのが魅力です。  
たとえば、入力された件数に応じて配列を作ることができます。

```c
int n = 100;
int *data = malloc((size_t)n * sizeof(int));
```

ただし、動的メモリは便利な反面、解放漏れや二重解放の危険があります。  
使う理由が明確なときにだけ使うと、無駄な複雑さを避けやすくなります。

## 10.7 第10章のまとめ

- C はメモリを自分で扱う場面が多いです
- 自動・静的・動的の 3 つをまず意識しましょう
- `malloc` と `free` は対で考えます
- 境界外アクセスは非常に危険です
