# chapter 11 文字列操作

## 11.1 文字列という「文化」

C の文字列は、ほかの多くの言語と比べると少し独特です。  
文字列は専用の豪華な型というより、`char` の並びとして表現されます。この文化を理解すると、C の文字列処理がぐっと見やすくなります。

## 11.2 文字列リテラル

`"Hello"` のような書き方を文字列リテラルといいます。  
これは、プログラムに直接書いた文字列です。

```c
printf("%s\n", "Hello");
```

`%s` は文字列を表示するための書式です。  
文字列の末尾には `'\0'` が入っていて、それが終わりを示します。

## 11.3 文字列の受け渡し手段

関数に文字列を渡すときは、`char *` や `char[]` を使います。

```c
void greet(const char *name) {
    printf("Hello, %s\n", name);
}
```

`const` をつけると、この関数は受け取った文字列を書き換えない、という約束になります。  
読むだけのときは `const` を積極的に使うと安全です。

## 11.4 文字列の実現手段

文字列を入れる場所としては、次の 2 つがよく使われます。

- 文字配列に直接入れる
- 動的に確保して使う

```c
char buf[32] = "abc";
```

このとき `buf` の中身は、`a`、`b`、`c`、`'\0'` です。  
文字列の長さより十分に大きい領域を用意しないと、あふれてしまいます。

## 11.5 文字列とオーバーラン

文字列の扱いで特に注意したいのが、バッファオーバーランです。  
用意した箱より多く書き込んでしまうと、別のデータを壊します。

```c
char buf[4] = "abc";
/* ここに長すぎる文字列を入れるのは危険 */
```

入力やコピーでは、サイズを意識した安全な関数を選ぶことが大切です。  
`gets` のような危険な関数は使わず、`fgets` や `snprintf` のような関数を使う考え方が基本です。

## 11.6 文字列を扱う標準関数

`<string.h>` には、よく使う文字列関数があります。

- `strlen`: 長さを調べる
- `strcmp`: 比較する
- `strcpy`: コピーする
- `strncpy`: 長さを指定してコピーする
- `strcat`: 連結する
- `strncmp`: 一定長で比較する

ただし、コピー系は使い方に注意が必要です。  
サイズを確認しながら使わないと、安全とは言えません。

## 11.7 文字列の配列

文字列をたくさん並べたいときは、文字列の配列を使います。

```c
const char *words[] = {"cat", "dog", "bird"};
```

これは、単語の一覧やメニュー表示などに便利です。  
ポインタの配列として理解すると、あとで応用しやすくなります。

## 11.8 第11章のまとめ

- C の文字列は `char` の並びです
- 末尾の `'\0'` がとても重要です
- `const char *` は読み取り専用の文字列を表すのに便利です
- 文字列操作では、サイズを必ず意識しましょう
