# chapter 12 パズルRPGの製作

## 12.1 「7つの約束」との別れ

ここからは、学んだ知識を使って小さな作品を作ります。  
本格的なゲームをいきなり完成させるのではなく、**動く最小のゲーム**を作るのが目標です。C では、機能を少しずつ積み上げていく感覚がとても大切です。

## 12.2 製作の流れ

パズル RPG は、たとえば次のような流れで作れます。

1. プレイヤーの状態を構造体で表す
2. マップを配列で持つ
3. 移動のルールを関数に分ける
4. 戦闘やイベントを関数で追加する
5. 最後に画面表示を整える

この順に作ると、完成を急がずに少しずつ進められます。  
最初は見た目より、**ルールが正しく動くこと**を優先しましょう。

## 12.3 ゲームの基本仕様

例として、次のようなシンプルな仕様を考えます。

- プレイヤーはマップ上を上下左右に移動できる
- アイテムを拾うと点数が増える
- 特定の場所で敵と出会う
- HP が 0 になったらゲームオーバー

この程度でも、配列、構造体、条件分岐、繰り返し、関数、文字列の基本を総合的に使えます。

```c
typedef struct {
    int x;
    int y;
    int hp;
    int item_count;
} Player;
```

## 12.4 パズルRPG製作課題

ここでは、完成品の答えを丸ごと与えるよりも、実装の骨組みを示します。

```c
void move_player(Player *p, char command) {
    switch (command) {
        case 'w': p->y--; break;
        case 's': p->y++; break;
        case 'a': p->x--; break;
        case 'd': p->x++; break;
        default: break;
    }
}
```

`->` は、ポインタで指した構造体のメンバにアクセスするときの書き方です。  
`p->x` は、`(*p).x` と同じ意味です。

ゲームづくりでは、**見た目が地味でも、ロジックが分かれていれば成功**です。  
マップ表示、入力処理、状態更新を分けると、あとで修正しやすくなります。
