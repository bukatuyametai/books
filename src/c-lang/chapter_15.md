# chapter 15 ツールによる効率化と安全なコード

## 15.1 道具による効率化

プログラミングは、コードを書くだけではありません。  
むしろ、**正しく作るための道具を使いこなすこと**が、実用開発ではとても大切です。

## 15.2 シェルスクリプト

同じコマンドを毎回手で打つのは大変です。  
シェルスクリプトにしておくと、ビルドや実行をまとめられます。

```bash
#!/bin/sh
gcc -std=c23 -Wall -Wextra main.c util.c -o app
./app
```

小さな自動化でも、毎日の手間を大きく減らしてくれます。

## 15.3 makeとMakefile

`make` は、変更された部分だけを再ビルドするための道具です。  
プロジェクトが大きくなるほど、そのありがたみが増します。

```make
app: main.o util.o
	gcc -std=c23 -Wall -Wextra main.o util.o -o app

main.o: main.c util.h
	gcc -std=c23 -Wall -Wextra -c main.c

util.o: util.c util.h
	gcc -std=c23 -Wall -Wextra -c util.c
```

「何を作るか」と「何に依存しているか」を明示できるのが利点です。

## 15.4 Doxygenの利用

Doxygen のような文書生成ツールを使うと、コメントから API 文書を作れます。  
関数の役割、引数、戻り値をきちんと書く習慣は、チーム開発で特に重要です。

```c
/**
 * 2つの整数を足して返す
 */
int add(int a, int b);
```

## 15.5 テストと静的解析

今の開発では、テストと静的解析がとても重要です。  
手で動かして確認するだけでなく、コードを機械にチェックさせることで、ミスを早く見つけられます。

- コンパイラ警告を有効にする
- テストを書いて繰り返し確認する
- 静的解析ツールで危険な書き方を見つける
- 必要に応じてサニタイザを使う

```bash
gcc -std=c23 -Wall -Wextra -fsanitize=address,undefined main.c -o app
```

こうした道具は、単に「上級者向けの装備」ではありません。  
むしろ、**最初から使うほど学習にも役立つ**道具です。

## 15.6 第15章のまとめ

- 開発はコードだけでなく道具の使い方も重要です
- `make` は再ビルドを効率化します
- Doxygen は文書化を助けます
- テスト、警告、静的解析は安全なコードの味方です

## 15.7 練習問題

1. いつも使うコンパイルコマンドをシェルスクリプトにしてみましょう。  
2. 簡単な Makefile を書いてみましょう。  
3. `-Wall -Wextra` をつけて出る警告を 1 つ直してみましょう。
