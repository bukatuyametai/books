# chapter 16 まだまだ広がるC言語の世界

## 16.1 C言語の可能性

C は入門しやすいだけでなく、広い世界へつながっています。  
この本で学んだことは、C そのものだけでなく、ほかの多くの言語やシステム開発にも役立ちます。変数、型、関数、配列、ポインタ、ファイル、ビルド。これらは多くの環境で共通する考え方です。

## 16.2 データベースの操作

C からデータベースを扱うには、専用のライブラリを使うのが一般的です。  
重要なのは、SQL を直接書くことそのものより、**外部の機能と連携する考え方**を身につけることです。実用開発では、C 単体ではなくライブラリと組み合わせて使う場面が多くあります。

## 16.3 ウィンドウアプリケーションの作成

C でも、GUI を使ったアプリケーションは作れます。  
ただし、初学者の段階では画面よりも、まずコンソールプログラムで基礎を固めるほうが学びやすいでしょう。イベント処理やウィンドウシステムは、その後に学ぶと理解が進みやすくなります。

## 16.4 インターネットへのアクセス

ネットワーク通信も C で行えます。  
ソケットや HTTP ライブラリを使えば、通信するプログラムを作れます。ここでも大切なのは、文字列やファイル、エラー処理の基礎がそのまま役立つことです。

## 16.5 Webアプリケーションの作成

Web アプリケーションの開発では、C 単体よりも Web サーバーやバックエンドライブラリと組み合わせることが多いです。  
C は、性能が必要な部品や土台の部分で強みを発揮します。見える部分を担当する言語と、速さが求められる部分を担当する C が、役割分担することもあります。

## 16.6 C言語を学び終えて

C を学ぶことは、単に古い言語を覚えることではありません。  
「コンピュータはどのようにデータを持ち、どのように処理し、どのように壊れるのか」を学ぶことでもあります。これは、どんな言語を学ぶときにも役立つ土台です。

この本を終えたとき、すべてを完全に理解していなくてもかまいません。  
大切なのは、次の 3 つができるようになることです。

- C のコードを読んで、だいたいの意味をつかめる
- 自分で小さなプログラムを書いて動かせる
- エラーが出ても、落ち着いて原因を探せる

ここまで来たなら、あなたはもう C 言語の入口をしっかりくぐっています。

## 16.7 本書の終わりに

最後まで読み進めたあなたは、もはや「まったくの初心者」ではありません。  
まだ学ぶことはたくさんありますが、それは良いことです。C は、学ぶほどに理解が積み重なる言語です。ここから先は、より大きなプログラム、より安全な書き方、より実践的な道具へと進んでいけます。

どうか、すぐに完璧を目指しすぎず、少しずつ書いて、少しずつ直して、少しずつ理解を深めてください。C の学習でいちばん大切なのは、**あきらめずに手を動かすこと**です。
