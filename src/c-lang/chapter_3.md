# chapter 3 式と演算子

## 3.1 計算の文

C では、値を計算して変数に入れることがよくあります。

```c
int a = 10;
int b = 3;
int c = a + b;
```

`a + b` のようなまとまりを式と呼びます。  
式は、値を作るための書き方です。式を使って、足し算、引き算、比較、代入などを行います。

## 3.2 オペランド

演算子は、`+` や `-` のような計算記号です。  
演算子の両側にある値や変数をオペランドと呼びます。

```c
int result = 7 + 5;
```

ここでは `7` と `5` がオペランド、`+` が演算子です。  
C の式では、どの順に計算されるかが大切です。

## 3.3 評価のしくみ

`3 + 4 * 5` は `35` ではなく `23` になります。  
これは、`*` のほうが `+` より先に計算されるからです。こうした優先順位を演算子の優先順位といいます。

```c
int x = 3 + 4 * 5;   /* 23 */
int y = (3 + 4) * 5; /* 35 */
```

迷ったら括弧を使うのが安全です。  
プログラムは人間の直感よりも文法を優先するので、**括弧で意図をはっきりさせる**ことが重要です。

## 3.4 演算子

よく使う演算子は次のとおりです。

- 算術演算子: `+ - * / %`
- 代入演算子: `=`
- 比較演算子: `== != < <= > >=`
- 論理演算子: `&& || !`
- インクリメント/デクリメント: `++ --`

とくに注意したいのは、`=` と `==` の違いです。

```c
int a = 10;      /* 代入 */
if (a == 10) {   /* 比較 */
    /* ... */
}
```

`=` は「入れる」、`==` は「等しいかどうかを調べる」です。  
この取り違えは初学者の代表的なミスなので、特に気をつけましょう。

`/` は整数同士だと小数部分が切り捨てられます。

```c
int a = 7 / 2;       /* 3 */
double b = 7 / 2.0;  /* 3.5 */
```

## 3.5 型の変換

計算では、型の違いにより結果が変わることがあります。  
整数と小数を混ぜると、より広い型に合わせて計算されることが多いですが、思った通りでない場合もあります。

```c
int a = 5;
double b = 2.0;
double c = a / b;  /* 2.5 */
```

必要に応じて、明示的に型変換を行うこともできます。

```c
int a = 7;
int b = 2;
double c = (double)a / b;  /* 3.5 */
```

## 3.6 命令実行の文

式をただ書くだけではなく、条件に応じて処理を変える場面もあります。  
その入口になるのが次章の条件分岐です。ここでは、式が「値を作る」、文が「処理を進める」という違いを押さえておきましょう。

## 3.7 第3章のまとめ

- 式は値を作ります
- 演算子には優先順位があります
- `=` と `==` は全く違います
- 整数の割り算では小数部分が切り捨てられます
