# chapter 4 条件分岐と繰り返し

## 4.1 プログラムの流れ

これまでは、上から下へ順番に実行されるコードを見てきました。  
しかし実際のプログラムでは、条件によって処理を変えたり、同じ処理を何度もくり返したりします。

- 条件に応じて道を分けるのが分岐
- ある処理を何回も行うのが繰り返し

これらをまとめて制御構文と呼びます。プログラムの流れを制御する構文だからです。

## 4.2 ブロックの書き方

条件分岐や繰り返しでは、`{}` によるブロックがよく登場します。

```c
if (score >= 60) {
    printf("合格です\n");
} else {
    printf("不合格です\n");
}
```

ブロックは、複数の文をひとまとまりとして扱うための入れ物です。  
`{}` が 1 つの文を表すこともあります。つまり、`if` の後に書く「実行する範囲」をまとめる役割です。

## 4.3 条件式の書き方

`if` では、条件式が真なら前半、偽なら `else` 側が実行されます。

```c
int age = 20;

if (age >= 18) {
    printf("大人です\n");
}
```

C では、0 は偽、それ以外は真として扱われます。  
とはいえ、初心者のうちは「比較の結果をそのまま条件に使う」書き方を優先するとわかりやすいです。

## 4.4 第4章のまとめ

- 分岐は、条件で処理を切り替えます
- 繰り返しは、同じ処理を何度も行います
- `if`、`else`、`while`、`for` は基本の制御構文です
- ブロックは `{}`
  で複数の文をまとめます
