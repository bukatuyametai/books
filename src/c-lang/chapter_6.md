# chapter 6 構造体

## 6.1 実用的な開発に必要なもの

プログラムが少し大きくなると、単独の数値だけでは表しにくい情報が増えてきます。  
たとえば「ユーザー」には名前、年齢、点数、会員状態など、いくつもの情報があります。こうした関連する値をひとまとめにするのが構造体です。

## 6.2 構造体とは

構造体は、複数の異なる型の値を 1 つにまとめる仕組みです。

```c
struct Student {
    char name[32];
    int age;
    double score;
};
```

この例では、`Student` という型に、名前・年齢・点数をまとめています。  
配列が「同じ型を並べる」のに対して、構造体は「異なる型を組み合わせる」ことができます。

## 6.3 構造体の使い方

構造体の値は、次のように作って使います。

```c
struct Student s1 = {"Taro", 20, 87.5};

printf("%s %d歳 %.1f点\n", s1.name, s1.age, s1.score);
```

メンバには `.` を使ってアクセスします。  
構造体を使うと、関連する情報を 1 つの名前でまとめて扱えるので、コードの意味が見えやすくなります。

## 6.4 構造体宣言のテクニック

毎回 `struct Student` と書くのは少し長いので、`typedef` を使って型名を短くすることがあります。

```c
typedef struct {
    char name[32];
    int age;
    double score;
} Student;
```

これで、`Student s1;` と書けるようになります。  
この書き方はよく使われますが、最初は「構造体の型を短く書ける便利な仕組み」として理解すれば十分です。

## 6.5 構造体の注意点

構造体の中には、配列や別の構造体も入れられます。  
ただし、値のコピーや代入の意味を理解しておく必要があります。

```c
Student a = {"Hanako", 19, 91.0};
Student b = a;   /* 全体がコピーされる */
```

構造体は便利ですが、データが増えると設計が大切になります。  
「何を 1 つのまとまりとして扱うのか」を意識すると、無理のない構造体を作れます。

## 6.6 第6章のまとめ

- 構造体は、関連する複数の値をひとまとめにする仕組みです
- `.` でメンバにアクセスします
- `typedef` を使うと型名を短くできます
- 構造体はコピーの意味も意識して使います

## 6.7 練習問題

1. `Book` という構造体を作り、題名・著者・価格を持たせてみましょう。  
2. 構造体の値を 1 つ作り、メンバを表示してみましょう。  
3. 2 つの構造体変数を代入して、コピーされることを確認してみましょう。
