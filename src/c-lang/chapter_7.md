# chapter 7 配列

## 7.1 配列とは

配列は、**同じ型の値を連続して並べる**ための仕組みです。  
たとえば、テストの点数 5 人分をまとめて持ちたいときに使います。

```c
int scores[5] = {80, 72, 90, 65, 88};
```

配列の要素には、0 から始まる番号でアクセスします。

```c
printf("%d\n", scores[0]);  /* 80 */
printf("%d\n", scores[4]);  /* 88 */
```

## 7.2 配列の準備

配列は、最初に要素数を決めるのが基本です。  
C の配列は、後からサイズを自由に変えられるわけではありません。

```c
int data[3];
```

この `data` には 3 個の `int` を入れられます。  
配列の大きさを超えて書き込むと、別の領域を壊してしまう危険があります。これは非常に重要です。

## 7.3 配列変数と要素の型

配列の中身は、すべて同じ型です。  
`int` の配列には `int` だけが入りますし、`char` の配列には `char` だけが入ります。

```c
char letters[3] = {'A', 'B', 'C'};
```

配列は「同じ種類のものをたくさん持つ」ための道具です。  
構造体が「異なる種類のものをまとめる」のに対し、配列は「同じ種類を並べる」と覚えると整理しやすくなります。

## 7.4 配列の使い方

配列を扱うときは、`for` 文との相性が抜群です。

```c
int scores[5] = {80, 72, 90, 65, 88};

for (int i = 0; i < 5; i++) {
    printf("%d\n", scores[i]);
}
```

要素番号は 0 から始まるため、`i < 5` のように書きます。  
`i <= 5` としてしまうと、最後に 1 つはみ出してしまいます。

## 7.5 配列の応用

配列は、表や一覧のデータにも使えます。  
さらに、2 次元配列を使うと、行と列を持つ表を表せます。

```c
int table[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};
```

これは小さな表計算やマップデータの基礎になります。  
ゲーム、画像処理、統計など、配列はさまざまな分野の土台です。

## 7.6 配列の注意点

配列の大きさと使う範囲を常に意識しましょう。  
C では、配列の端をはみ出しても自動で止まってはくれません。

```c
int a[3] = {1, 2, 3};
/* a[3] は範囲外 */
```

範囲外アクセスは、見つかるとは限らない不具合を引き起こします。  
「たまたま動く」は安全ではありません。

## 7.7 配列と文字列型

C では、文字列は `char` の配列として扱います。  
この関係を理解すると、`printf("%s", ...)` や文字列操作がわかりやすくなります。

```c
char msg[] = "Hello";
```

文字列については、次のポインタとあわせて学ぶとさらに理解が深まります。

## 7.8 第7章のまとめ

- 配列は、同じ型の値を並べます
- 要素番号は 0 から始まります
- 範囲外アクセスは非常に危険です
- C の文字列は `char` 配列として表されます
