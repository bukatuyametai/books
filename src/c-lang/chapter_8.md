# chapter 8 関数

## 8.1 関数とは

関数は、**まとまった処理に名前をつけたもの**です。  
同じ処理を何度も書かずに、1 つにまとめて呼び出せるのが大きな利点です。

```c
#include <stdio.h>

void hello(void) {
    printf("こんにちは\n");
}
```

この `hello` は、何も受け取らず、何も返さない関数です。

## 8.2 引数の利用

関数は、値を受け取って処理できます。受け取る値を引数といいます。

```c
int add(int a, int b) {
    return a + b;
}
```

呼び出す側は、次のように使います。

```c
int x = add(3, 4);  /* 7 */
```

引数を使うと、同じ処理をさまざまな値に対して使い回せます。  
「3 と 4 を足す関数」ではなく、「2 つの値を足す関数」になるわけです。

## 8.3 戻り値の利用

関数は、処理した結果を返せます。

```c
double area(double radius) {
    return 3.141592653589793 * radius * radius;
}
```

`return` は、関数の実行を終え、値を返します。  
戻り値を持つ関数では、返す型を最初に書きます。

## 8.4 関数とスコープ

関数の中で宣言した変数は、その関数の外からは見えません。  
これをスコープと呼びます。

```c
void f(void) {
    int x = 10;
}
/* x はここでは使えない */
```

スコープを理解すると、変数がどこで生きているのかが整理できます。  
似た名前の変数を別々の場所で使うときも、スコープが助けになります。

## 8.5 身近な関数たち

これまで使ってきた `printf` も関数です。  
C では、標準ライブラリにたくさんの便利な関数があります。

- `printf`: 画面に表示する
- `scanf`: 入力を読む
- `strlen`: 文字列の長さを調べる
- `strcmp`: 文字列を比較する

## 8.6 関数の活用と壁

関数を増やすと、プログラムは読みやすく、直しやすくなります。  
しかし、関数を増やしすぎて何をしているかわからなくなることもあります。大切なのは、**1 つの関数が 1 つの役割を持つ**ように整理することです。

また、C では関数の宣言と定義の扱いにも注意が必要です。  
この話は、複数ファイルで開発する第 13 章でしっかり扱います。

## 8.7 第8章のまとめ

- 関数は処理をまとめて名前をつけたものです
- 引数で値を受け取り、戻り値で結果を返せます
- スコープは変数の見える範囲です
- 小さく役割のはっきりした関数が読みやすさを生みます
