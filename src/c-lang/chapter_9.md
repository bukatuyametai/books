# chapter 9 アドレスとポインタ

## 9.1 C言語だけが持つ特徴

C を学ぶ上で、多くの人が最初につまずくのがポインタです。  
でも、ポインタは「難しい魔法」ではありません。ひと言でいえば、**メモリ上の場所を示す値**です。

ポインタを理解すると、C がなぜ強力なのかが見えてきます。  
同時に、なぜ危険もあるのかがわかります。

## 9.2 メモリ

プログラムは、実行時にメモリを使います。  
変数や配列、文字列などは、どこかのメモリ領域に置かれます。

```c
int x = 10;
```

この `x` は、メモリのある場所に保存されています。  
ポインタは、その場所を指し示すための値です。

## 9.3 アドレスの取得

`&` を使うと、変数のアドレスを取得できます。

```c
int x = 10;
printf("%p\n", (void*)&x);
```

ここで `&x` は、`x` の置かれている場所です。  
`%p` はアドレス表示に使う書式指定子です。実際には `void*` に変換して表示するのが一般的です。

## 9.4 アドレスの解決

ポインタが指している先の値を取り出すには、`*` を使います。  
これを間接参照といいます。

```c
int x = 10;
int *p = &x;

printf("%d\n", *p);  /* 10 */
```

`p` には `x` の場所が入っています。`*p` と書くと、その場所にある値を読み書きできます。  
この「場所を持つ値」という感覚が、ポインタ理解の第一歩です。

## 9.5 ポインタを使うメリット

ポインタが役立つ理由はいくつもあります。

- 大きなデータを関数に渡しやすい
- 配列や文字列を効率よく扱える
- メモリを直接意識した柔軟な処理ができる

たとえば、関数の中で変数を書き換えたいとき、値そのものではなくアドレスを渡すことがあります。

```c
void set_zero(int *p) {
    *p = 0;
}
```

呼び出し側では次のようにします。

```c
int x = 5;
set_zero(&x);
```

関数の中から、元の変数を直接変更できます。

## 9.6 第9章のまとめ

- ポインタはメモリ上の場所を示します
- `&` でアドレスを取り、`*` で中身にアクセスします
- ポインタは関数と配列を扱うときに重要です
- 便利ですが、危険もあるので丁寧に使う必要があります

3. ポインタを受け取って値を書き換える関数を書いてみましょう。
